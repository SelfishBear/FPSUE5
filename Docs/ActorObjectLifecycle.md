# Жизненный цикл AActor и UObject — полный справочник

---

## Часть 1: UObject — базовый жизненный цикл

`UObject` — корень всего в Unreal. У него минимальный набор методов.

```
 ┌─────────────────────────────────────┐
 │  CDO создаётся при загрузке модуля  │  (Class Default Object)
 └──────────────┬──────────────────────┘
                ▼
 ┌──────────────────────────┐
 │  Constructor (C++)       │  Вызывается движком, НЕ вызывай NewObject в конструкторе
 └──────────────┬───────────┘
                ▼
 ┌──────────────────────────┐
 │  PostInitProperties()    │  Свойства инициализированы дефолтами из CDO
 └──────────────┬───────────┘
                ▼
 ┌──────────────────────────┐
 │  PostLoad()              │  Только для объектов загруженных с диска (уровень/ассет)
 └──────────────┬───────────┘
                ▼
 ┌──────────────────────────┐
 │  Объект живёт и работает │
 └──────────────┬───────────┘
                ▼
 ┌──────────────────────────┐
 │  BeginDestroy()          │  Первый шаг уничтожения — освободи свои ресурсы
 └──────────────┬───────────┘
                ▼
 ┌──────────────────────────┐
 │  IsReadyForFinishDestroy │  GC спрашивает: "Можно завершить?" (return true/false)
 └──────────────┬───────────┘
                ▼
 ┌──────────────────────────┐
 │  FinishDestroy()         │  Финальная очистка, после этого память освобождается
 └──────────────────────────┘
```

### Краткая таблица UObject

| Метод | Когда вызывается | Типичное использование |
|-------|-----------------|----------------------|
| `Constructor` | При создании CDO и каждого экземпляра | Установить дефолтные значения примитивов |
| `PostInitProperties()` | После конструктора, свойства заполнены | Инициализация зависящая от UPROPERTY значений |
| `PostLoad()` | После десериализации с диска | Миграция/фикс старых данных ассета |
| `BeginDestroy()` | GC решил уничтожить объект | Освободить ресурсы, отписаться от делегатов |
| `FinishDestroy()` | Последний шаг перед освобождением памяти | Низкоуровневая очистка (редко нужен) |

## Часть 2: AActor — полный жизненный цикл

`AActor` наследует `UObject` и добавляет свои этапы.

### 2.1 Порядок при спавне в мире

```
 ┌───────────────────────────────────────────────────┐
 │  1. Constructor (C++)                             │
 │     • Создаются компоненты через CreateDefaultSubobject
 │     • НЕ трогай мир, НЕ спавнь других акторов    │
 │     • Только дефолтные значения                   │
 └──────────────────┬────────────────────────────────┘
                    ▼
 ┌───────────────────────────────────────────────────┐
 │  2. PostInitProperties()                          │
 │     • Свойства из CDO скопированы                 │
 └──────────────────┬────────────────────────────────┘
                    ▼
 ┌───────────────────────────────────────────────────┐
 │  3. PostLoad() — ТОЛЬКО если актор загружен       │
 │     с уровня (не SpawnActor)                      │
 └──────────────────┬────────────────────────────────┘
                    ▼
 ┌───────────────────────────────────────────────────┐
 │  4. OnConstruction(FTransform)                    │
 │     • Construction Script в Blueprint             │
 │     • Вызывается в редакторе при перемещении      │
 │     • Вызывается при SpawnActor                   │
 └──────────────────┬────────────────────────────────┘
                    ▼
 ┌───────────────────────────────────────────────────┐
 │  5. PostInitializeComponents()                    │
 │     • Все компоненты актора инициализированы      │
 │     • Можно безопасно обращаться к компонентам    │
 └──────────────────┬────────────────────────────────┘
                    ▼
 ┌───────────────────────────────────────────────────┐
 │  6. BeginPlay()                                   │
 │     • Мир полностью загружен и готов              │
 │     • Безопасно искать других акторов             │
 │     • Подписываться на делегаты                   │
 │     • Запускать таймеры                           │
 │     • Blueprint: Event BeginPlay                  │
 └──────────────────┬────────────────────────────────┘
                    ▼
 ┌───────────────────────────────────────────────────┐
 │  7. Tick(float DeltaTime)                         │
 │     • Каждый кадр (если включён)                  │
 │     • PrimaryActorTick.bCanEverTick = true        │
 └──────────────────┬────────────────────────────────┘
                    ▼
 ┌───────────────────────────────────────────────────┐
 │  8. EndPlay(EEndPlayReason)                       │
 │     • Актор удаляется / уровень выгружается       │
 │     • Отписка от делегатов, остановка таймеров    │
 └──────────────────┬────────────────────────────────┘
                    ▼
 ┌───────────────────────────────────────────────────┐
 │  9. Destroyed()                                   │
 │     • Вызывается после Destroy()                  │
 │     • OnDestroyed делегат                         │
 └──────────────────┬────────────────────────────────┘
                    ▼
 ┌───────────────────────────────────────────────────┐
 │  10. BeginDestroy() → FinishDestroy()             │
 │     • Стандартный цикл GC от UObject              │
 └───────────────────────────────────────────────────┘
```

### 2.2 Таблица AActor

| # | Метод | Когда | Что делать |
|---|-------|-------|-----------|
| 1 | `Constructor` | Создание объекта | `CreateDefaultSubobject`, дефолтные значения |
| 2 | `PostInitProperties` | После конструктора | Редко переопределяется в акторе |
| 3 | `PostLoad` | Загрузка с диска | Миграция данных |
| 4 | `OnConstruction` | После спавна / в редакторе | Процедурная генерация, Construction Script |
| 5 | `PostInitializeComponents` | Компоненты готовы | Инициализация связей между компонентами |
| 6 | **`BeginPlay`** | Мир готов, игра начинается | **Основная точка инициализации** |
| 7 | `Tick` | Каждый кадр | Логика обновления |
| 8 | `EndPlay` | Актор покидает мир | Очистка: таймеры, делегаты |
| 9 | `Destroyed` | После вызова `Destroy()` | Спавн эффектов смерти, оповещение |
| 10 | `BeginDestroy` | GC чистит память | Низкоуровневая очистка |

## Часть 3: UActorComponent — жизненный цикл компонентов

```
 Constructor
     ▼
 InitializeComponent()      ← если bWantsInitializeComponentEvent = true
     ▼
 BeginPlay()                ← после BeginPlay актора-владельца
     ▼
 TickComponent(DeltaTime)   ← каждый кадр
     ▼
 EndPlay()
     ▼
 UninitializeComponent()
     ▼
 OnComponentDestroyed()
     ▼
 BeginDestroy()
```

| Метод | Что делать |
|-------|-----------|
| `InitializeComponent()` | Ранняя инициализация, до BeginPlay |
| `BeginPlay()` | Основная инициализация |
| `TickComponent()` | Логика каждый кадр |
| `EndPlay()` | Очистка |
| `UninitializeComponent()` | Отмена регистрации |
| `OnComponentDestroyed()` | Финальная очистка |

## Часть 4: ACharacter / APawn — дополнительные методы

`ACharacter` наследует `APawn` → `AActor`, добавляет:

| Метод | Когда | Что делать |
|-------|-------|-----------|
| `PossessedBy(AController*)` | Контроллер захватывает пешку | Инициализация управления (сервер) |
| `UnPossessed()` | Контроллер отпускает пешку | Очистка |
| `OnRep_Controller()` | Репликация контроллера (клиент) | Клиентская инициализация |
| `SetupPlayerInputComponent(UInputComponent*)` | Пешка готова получать инпут | Привязка действий ввода |
| `Restart()` | Пешка перезапускается | — |
| `PawnClientRestart()` | Клиентский рестарт | Enhanced Input здесь часто настраивают |

### Порядок для ACharacter при спавне

```
Constructor
    ▼
PostInitializeComponents()
    ▼
PossessedBy(Controller)         ← контроллер захватывает
    ▼
SetupPlayerInputComponent()     ← привязка ввода
    ▼
BeginPlay()
    ▼
Tick() каждый кадр
    ▼
UnPossessed()                   ← если контроллер отпускает
    ▼
EndPlay()
    ▼
Destroyed()
```

## Часть 6: UUserWidget (UI) — жизненный цикл

```
Constructor (C++)
    ▼
Initialize()                    ← Виджет создан в памяти
    ▼
NativeConstruct()               ← Виджет добавлен в viewport (аналог BeginPlay)
    ▼
NativeTick(FGeometry, DeltaTime)
    ▼
NativeDestruct()                ← Виджет убран из viewport
    ▼
RemoveFromParent()              ← Удаление из иерархии
    ▼
BeginDestroy()                  ← GC
```

| Метод | Blueprint-аналог | Когда |
|-------|-----------------|-------|
| `Initialize()` | — | Виджет создан, но ещё не на экране |
| `NativeConstruct()` | `Event Construct` | Добавлен в viewport |
| `NativeTick()` | `Event Tick` | Каждый кадр |
| `NativeDestruct()` | `Event Destruct` | Убран из viewport |

## Часть 7: GameMode / GameState — порядок загрузки уровня

```
 1. UEngine::Init()
 2. UWorld создаётся
 3. AGameModeBase::InitGame()               ← GameMode инициализируется
 4. Все акторы на уровне: Constructor
 5. Все акторы: PostInitializeComponents()
 6. AGameModeBase::PreLogin()               ← Проверка подключения игрока
 7. AGameModeBase::Login()                  ← Создаётся PlayerController
 8. AGameModeBase::PostLogin()              ← PlayerController готов
 9. AGameModeBase::HandleStartingNewPlayer()← Спавн пешки
 10. APawn::PossessedBy()
 11. Все акторы: BeginPlay()                ← МИР ГОТОВ
 12. AGameModeBase::StartPlay()
```

## Часть 8: Создание через SpawnActor vs размещение на уровне

### Размещён на уровне (Level Placed)
```
Constructor → PostInitProperties → PostLoad → OnConstruction →
PostInitializeComponents → BeginPlay
```

### SpawnActor в рантайме
```
Constructor → PostInitProperties → OnConstruction →
PostInitializeComponents → BeginPlay
```
> **Разница:** `PostLoad` вызывается ТОЛЬКО для объектов загруженных с диска.

### NewObject\<T\> (для UObject, не AActor)
```
Constructor → PostInitProperties → [готов к использованию]
```
> **Нет BeginPlay!** UObject не имеет BeginPlay. Это метод AActor.

## Часть 9: Шпаргалка — где что инициализировать

| Задача | Где делать | Почему |
|--------|-----------|--------|
| Создать компонент | `Constructor` | `CreateDefaultSubobject` работает только здесь |
| Установить дефолтные значения | `Constructor` | CDO копирует их |
| Обратиться к другим акторам | `BeginPlay` | До BeginPlay мир не гарантированно готов |
| Подписаться на делегат | `BeginPlay` | Объекты гарантированно существуют |
| Запустить таймер | `BeginPlay` | World и TimerManager доступны |
| Привязать ввод | `SetupPlayerInputComponent` | Специальный метод для этого |
| Отписаться от делегатов | `EndPlay` | Гарантированно вызовется |
| Остановить таймеры | `EndPlay` | Последний шанс до уничтожения |
| Спавнить VFX при смерти | `Destroyed` или `EndPlay` | Актор ещё в мире |

## Часть 10: Частые ошибки

| Ошибка | Проблема | Решение |
|--------|---------|---------|
| `SpawnActor` в Constructor | Мир не существует → крэш | Перенести в `BeginPlay` |
| `GetWorld()` в Constructor | Возвращает `nullptr` | Использовать в `BeginPlay` |
| `FindComponentByClass` в Constructor | Компоненты не готовы | `PostInitializeComponents` или `BeginPlay` |
| Не вызвать `Super::BeginPlay()` | Компоненты не получат BeginPlay | Всегда вызывай `Super::` |
| Забыть `EndPlay` очистку | Утечки, крэши после смены уровня | Отписка и остановка в `EndPlay` |

## Быстрая visual-шпаргалка: AActor

```
CONSTRUCTOR          ← дефолты, компоненты
    ▼
OnConstruction       ← Construction Script
    ▼
PostInitComponents   ← компоненты готовы
    ▼
BeginPlay ★          ← ОСНОВНАЯ ТОЧКА: мир готов, делегаты, таймеры
    ▼
Tick                 ← каждый кадр
    ▼
EndPlay ★            ← ОЧИСТКА: отписки, таймеры стоп
    ▼
Destroyed            ← последний выдох
    ▼
GC убирает из памяти
```

> ★ — два самых важных метода которые ты будешь использовать в 90% случаев.
